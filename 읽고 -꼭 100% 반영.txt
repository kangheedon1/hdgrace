실행 가능한 상태로 바스29.0.0 버젼 에서 정상 작동 하도록 리팩토링해야 합니다.  

=== 리팩토링 원칙 ===
1. 기존 기능 100% 보존

     태그 불일치/잘못된 시작 태그/닫힘 없는 태그 등 모든 오류를 수정해서 정상 작동하도록 복원해야 합니다.

   - 절대 주석 처리하거나 삭제하지 말고, 반드시 살려서 완전하게 동작하도록 보완해야 합니다.

1) 폴더 구조(권장)

project-root/
├─ project.xml                # BAS 실전 통합 코드 (아래 템플릿)
├─ config/
│  ├─ prod.json               # 운영 설정(입출력 경로, 스레드, 정책)
│  └─ secrets.json            # 키/비번/토큰(하드코딩 금지, 여기서 로드)
├─ input/                     # 원본 파일/데이터 위치
├─ output/                    # 산출물(머지/로그/리포트)
├─ logs/                      # 실행/오류/감사 로그
└─ README.txt                 # 실행/복구/문제해결/AC 체크리스트

※ 운영에서 파일이 19MB↑면, 배포 산출물을 19MB 이하로 분할(zip) + 재조립 안내 포함.

2) 필수 체크(실전 요구사항 요약)

- BAS 버전: 29.00 (정확히 명시)
- 실행 모드: 병렬(Parallel) 기본, 스레드·딜레이·리소스 가드 명시
- 보안: 모든 민감정보는 secrets.json에서 로드(코드에 하드코딩 금지)
- 에러 처리: Try/Catch 또는 If/Then/Else + 재시도 + 백오프
- 로깅: 실행 로그(INFO/ERROR) + 감사 로그(상태변경) 분리 저장
- 중복 제거: “완전히 동일한 줄”만 제거(유사 문구는 유지)
- 테스트 코드: 파일명/주석에 test/demo/sample 있으면 제외(옵션)
- UI: 한국어 라벨 + 이모지 버튼(▶️ 실행, 🔎 검증, 🧾 로그)
- 문법 안정: 태그 닫힘, 속성 따옴표, 중첩 구조 점검
- 원본 라인 보존: 라벨/주석은 “앞줄”에만 추가(원본 변경 금지)
